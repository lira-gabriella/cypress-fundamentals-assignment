It is awesome that i discovered a lot in automation. Moving from just reading boring theory to actually watching Cypress hijack a live browser page was an amazing feeling for me.When I first started, commands like cy.visit() to open up the page, and chaining cy.get().type() to fill out form fields made total sense right away. It feels incredibly intuitive—almost like you are writing a normal sentence to tell the computer what to do. Seeing the browser automatically type out text and flip radio buttons with cy.check() on the first try was a massive confidence booster. I even figured out how to use cy.selectFile() to upload mock text files dynamically, and cy.scrollIntoView() to force the browser to scroll down so hidden footer elements wouldn't cause errors.The journey definitely wasn't without roadblocks, though. Dropdown assertions completely tripped me up at first. I spent way too much time debugging why a test failed, only to realize the visible text on screen said 'Canada' with a capital letter, but the underlying HTML attribute used lowercase values. Dealing with those minor text-casing differences and hunting through strict selector assertions really forced me to look closely at how the browser interprets the DOM tree.If you want to jump in, the setup is incredibly fast. You just need Node.js installed, clone the repository, and run npm install cypress --save-dev in your terminal. You can use npx cypress open to watch the tests run visually, or npx cypress run to blast through them silently in headless mode. Next up, I am super eager to learn API testing with cy.request() to bypass the UI entirely. You should absolutely try it—seeing your own code drive a browser automatically is the coolest thing!
