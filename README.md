Cypress Automation Practice SuiteObjectiveThis project marks my first real step into web automation using Cypress. The goal of this assignment was to move past just reading about testing commands and actually use them to control a live web page. By writing scripts to interact with a real practice website, I learned how Cypress finds elements, fills out user data, verifies page behaviors, and ensures that web components function exactly as expected.Explanation of Core CommandsWhen I first started, I learned that everything begins with cy.visit(). This is the command that actually boots up the browser and drives it to a specific web address. In my project, I placed it inside a before block so that Cypress opens up https://blogspot.com right away, giving all of my subsequent tests a live canvas to interact with.Once the page loaded, I realized I needed a way to pinpoint specific items on the screen, which is exactly what cy.get() does. It acts like a search tool that scans the underlying HTML code using CSS selectors like IDs or classes. For example, writing cy.get("#name") allowed my script to lock onto the exact name input field before trying to do anything else with it.After locking onto an element, I used cy.type() to actually enter data into the form. It mimics a real user typing on a keyboard letter by letter. I used this to fill out the form fields, like entering an email address with cy.get("#email").type("john.doe@example.com"), which showed me how easily Cypress handles user input.Handling drop-down menus was another major milestone, and I solved this using cy.select(). Instead of clicking blindly, this command specifically targets <select> dropdown elements to choose a precise option. In my script, cy.get("#country").select("Canada") automatically flipped the option to Canada, and I paired it with an assertion to verify that the internal value updated correctly.I also got to work with toggles using cy.check() and cy.uncheck(). These commands are specifically designed to interact with radio buttons and checkboxes. It was amazing to watch Cypress instantly click the male radio button using cy.get("#male").check(), and then chain actions together to check and uncheck the Sunday option to prove the checkbox could handle state changes.One of the coolest things I figured out was how to handle file uploads using cy.selectFile(). Usually, file inputs can be tricky to automate, but this command allowed me to create a mock text file buffer directly in the code named upload-manifest.txt and attach it to the page dynamically, verifying that the system successfully registers file attachments.Finally, I discovered that tests can fail if an element is hidden far down the page, which brought me to cy.scrollIntoView(). This command forces the browser viewport to scroll down until a target element is fully visible. I applied this to the footer section by running cy.contains("h2", "Footer Links").scrollIntoView(), ensuring my script could safely interact with elements at the very bottom of the page without running into visibility errors.Setup and Execution InstructionsTo get this project running locally, you just need to make sure Node.js is installed on your machine. First, open your terminal and clone this repository using git clone followed by your repository link. Once cloned, navigate into the project directory using cd your-project-folder. From there, install Cypress as a development dependency by running npm install cypress --save-dev.To see the tests run visually, open the interactive Cypress Test Runner by executing npx cypress open in your terminal. If you want to execute the tests quickly in the background without launching a browser UI, you can run them headlessly using npx cypress run instead.Personal ReflectionWhat was the easiest concept to understand?The easiest concept for me to grasp was combining cy.get() with cy.type(). The logic felt incredibly natural and intuitive—almost like writing a sentence. Telling the computer to "get this ID and type this text" worked perfectly on the first try, and seeing the browser immediately type out the text automatically was a huge confidence booster.What was the most challenging?The most challenging part was dealing with strict selector assertions, especially with the dropdown element. I ran into an issue where typing the visible text "Canada" didn't immediately pass until I realized the underlying HTML asset used lowercase letters for its value attribute. Debugging why a test failed over minor text casing differences and managing element visibility forced me to really look closely at how the browser interprets the DOM tree.What would you like to learn next about Cypress?Now that I have a solid handle on UI interactions, I am incredibly eager to learn how to do API testing using cy.request(). Bypassing the frontend interface to test server endpoints, validate status codes, and set up database states instantly sounds powerful. Learning how to blend API actions with UI tests seems like the ultimate way to build faster, rock-solid automation frameworks.
